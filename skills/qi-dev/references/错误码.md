# qi 框架预定义错误码（1000–1103）

> 业务错误码从 **2000+** 开始，避免冲突。

| 变量 | Code | HTTP Status | 默认消息 |
|------|------|-------------|----------|
| `qi.ErrServer` | 1000 | 500 | server error |
| `qi.ErrBadRequest` | 1001 | 400 | bad request |
| `qi.ErrUnauthorized` | 1002 | 401 | unauthorized |
| `qi.ErrForbidden` | 1003 | 403 | forbidden |
| `qi.ErrNotFound` | 1004 | 404 | not found |
| `qi.ErrConflict` | 1005 | 409 | conflict |
| `qi.ErrTooManyRequests` | 1006 | 429 | too many requests |
| `qi.ErrInvalidParams` | 1100 | 400 | invalid parameters |
| `qi.ErrMissingParams` | 1101 | 400 | missing parameters |
| `qi.ErrInvalidFormat` | 1102 | 400 | invalid format |
| `qi.ErrOutOfRange` | 1103 | 400 | out of range |

## 错误克隆链用法

```go
// 附加底层错误和自定义消息（返回新实例，不修改哨兵）
return nil, qi.ErrNotFound.
    WithErr(sql.ErrNoRows).
    WithMessagef("用户 %s 不存在", id)

// 仅修改消息
return nil, qi.ErrBadRequest.WithMessage("page 不能小于 1")

// 格式化消息
return nil, qi.ErrNotFound.WithMessagef("订单 %s 不存在", orderID)

// 修改 HTTP 状态码
return nil, qi.ErrBadRequest.WithStatus(http.StatusUnprocessableEntity)

// 业务包自定义错误
var ErrOrderTimeout = errors.NewWithStatus(3001, http.StatusGatewayTimeout, "订单超时")
```

## 错误工具函数

```go
import "github.com/tokmz/qi/pkg/errors"

// 包装任意 error 为业务错误
err := errors.Wrap(dbErr, 5001, "数据库查询失败")
err := errors.WrapWithStatus(dbErr, 5001, http.StatusInternalServerError, "数据库查询失败")

// 从 error 提取信息（兼容普通 error 和 *errors.Error）
code   := errors.GetCode(err)       // 非 *Error 返回 -1
status := errors.GetStatus(err)     // 非 *Error 返回 500

// 快速判断
if errors.IsCode(err, 1004) { /* 是 ErrNotFound */ }
if errors.Is(err, qi.ErrNotFound) { /* 标准库语义 */ }

// 类型断言
if qe, ok := errors.As(err); ok {
    fmt.Println(qe.Code, qe.Status())
}
```
